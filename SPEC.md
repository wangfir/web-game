# Pac-Man 吃豆人游戏规范

## 1. 项目概述

- **项目名称**: Classic Pac-Man
- **项目类型**: 2D 经典街机游戏网页版
- **核心功能**: 玩家控制吃豆人在迷宫中收集豆子，回避幽灵，吃能量药丸反杀幽灵
- **目标用户**: 休闲游戏玩家、怀旧游戏爱好者

## 2. 视觉与渲染规范

### 场景设置
- **画布尺寸**: 响应式，最大宽度 600px
- **背景色**: 深蓝色 `#0d0d2b`
- **迷宫风格**: 复古霓虹风格，蓝色墙壁配发光效果

### 角色设计
- **吃豆人**:
  - 经典黄色圆形，带嘴巴动画
  - 嘴巴张合频率: 4 次/秒
  - 移动方向跟随嘴巴朝向
  - 尺寸: 迷宫单元格的 80%

- **幽灵 (4只)**:
  - 经典幽灵造型，头顶波浪状
  - 颜色: 红色、粉色、青色、橙色
  - 尺寸: 迷宫单元格的 80%
  - 眼睛跟随移动方向

### 道具设计
- **普通豆子**: 小圆形，浅黄色，尺寸 8px，带微弱发光
- **能量药丸**: 大圆形，白色，尺寸 16px，脉冲动画
- **墙壁**: 蓝色 `#1e3a8a`，带圆角，内发光效果

### UI 设计
- **分数显示**: 左上角，白色像素字体风格
- **生命显示**: 左下角，吃豆人图标 × 剩余生命数
- **关卡显示**: 右上角
- **开始界面**: 居中标题 + "按空格开始" 提示
- **游戏结束/胜利界面**: 半透明遮罩 + 结果文字

### 视觉效果
- 吃豆人嘴巴动画 (开/合循环)
- 吃豆时的轻微放大动画
- 能量药丸脉冲效果
- 幽灵死亡时变成眼睛飘回出生点
- 墙壁发光边框效果

## 3. 游戏机制规范

### 迷宫设计
- 15×15 格子迷宫
- 上下左右移动
- 支持传送通道 (左右两侧)

### 移动系统
- 键盘控制: 方向键 或 WASD
- 吃豆人持续向当前方向移动
- 到达墙壁自动停止
- 缓冲区系统: 提前输入方向，在可行时立即转向

### 得分系统
- 普通豆子: 10 分/个
- 能量药丸: 50 分/个
- 吃幽灵: 200/400/800/1600 分 (连续翻倍)

### 敌人 AI
- **红幽灵 (Blinky)**: 追击模式，直奔玩家位置
- **粉幽灵 (Pinky)**: 包围模式，预测玩家前方 4 格
- **青幽灵 (Inky)**: 随机模式，随机移动
- **橙幽灵 (Clyde)**: 害羞模式，远离时追击，靠近时逃跑

### 能量模式
- 持续时间: 8 秒
- 幽灵变蓝、变慢
- 效果结束后幽灵变回原色并闪烁警告

### 游戏流程
- **开始**: 3 秒倒计时
- **死亡**: 失去一条命，1.5 秒后重生
- **胜利**: 吃完所有豆子，进入下一关
- **失败**: 3 条命用尽
- **难度递增**: 每关幽灵速度增加 5%

## 4. 交互规范

### 控制方式
- 键盘方向键: 上/下/左/右
- WASD 键: W 上/A 左/S 下/D 右
- 空格键: 开始游戏 / 重新开始

### 音效 (可选，静音默认)
- 背景音乐: 经典吃豆人 BGM
- 吃豆声
- 能量药丸声
- 死亡音效
- 胜利音效

## 5. 技术实现

- **技术栈**: 原生 HTML5 + Canvas + JavaScript
- **帧率**: 60 FPS
- **状态管理**: 菜单/游戏中/暂停/游戏结束/胜利
